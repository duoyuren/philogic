Positron connections pane

General tutorial

Real life use cases:

  • Mirror of a relational database of Idaho’s election results for testing and local development. It’s a core part of the DIY Election Desk project that I presented about at posit::conf(2025) (see code here)
  • Taking a massive 10 GB zipped CSV file and putting it in a DuckDB database and accessing it through {dbplyr} (see code here)

Example with DuckDB and {connections}; the same general process works with any other database backend, like SQLite, MySQL, Spark, PostgreSQL, etc.:

library(tidyverse)

# Use nicer DuckDB Connections Pane features
# options("duckdb.enable_rstudio_connection_pane" = TRUE)

# Connect to a database
db_file <- "my_fancy_database.duckdb"
con <- connections::connection_open(duckdb::duckdb(), db_file)

# Add stuff to it
copy_to(
  con,
  gapminder::gapminder,
  name = "gapminder",
  overwrite = TRUE,
  temporary = FALSE
)

# Get stuff out of it
gapminder_2007 <- tbl(con, I("gapminder")) |>
  filter(year == 2007) |>
  collect()

# All done
connections::connection_close(con)

Some things I do with it:

  • Typographic things like:

    • ;comm for ⌘, ;shift for ⇧, ;opt for ⌥
    • ;times for a true multiplication sign ×, ;minus for a true subtraction sign −
    • ;prime1 and ;prime2 for ′ and ″ (like 5′ 11″)

  • …and date things like:

    • ;dt for “2026-01-13”
    • ;dz for “2026-01-13T11:06”
    • ;dm for “2026-01-13 11:06:05”
    • ;date for “January 13, 2026”

  • …and longer words with repeated letters that I always mess up, like these:

    • insttn for “institution”
    • consttal for “constitutional”
    • entrepal for “entrepreneurial”

  • …and code snippets like ;dplyrsum for inserting this R code to disable {dplyr}’s grouping messages:

    options(dplyr.summarise.inform = FALSE)

  • …or running a {targets} pipeline with ;tarm:

    targets::tar_make()

  • …or adding Quarto chunk metadata like disabling warnings and messages with ;wm:

    #| warning: false

## Positron connections pane

[**General tutorial**](https://www.andrewheiss.com/blog/2025/07/10/positron-database-connections/)

Real life use cases:

- Mirror of a relational database of Idaho's election results for testing and local development. It's a core part of the [DIY Election Desk](https://andrewheiss.github.io/election-desk/) project that [I presented about at posit::conf(2025)](https://andrewheiss.github.io/election-desk/posit-conf-2025.html) ([see code here](https://github.com/andrewheiss/election-desk/blob/main/etl-pipeline/R/funs_db.R))
- Taking a massive 10 GB zipped CSV file and putting it in a DuckDB database and accessing it through {dbplyr} ([see code here](https://github.com/andrewheiss/tight-tortoise/blob/main/R/csvs-to-duckdb.R))

Example with DuckDB and [{connections}](https://rstudio.github.io/connections/); the same general process works with any other database backend, like SQLite, MySQL, Spark, PostgreSQL, etc.:

```{r}
#| eval: false

library(tidyverse)

# Use nicer DuckDB Connections Pane features
# options("duckdb.enable_rstudio_connection_pane" = TRUE)

# Connect to a database
db_file <- "my_fancy_database.duckdb"
con <- connections::connection_open(duckdb::duckdb(), db_file)

# Add stuff to it
copy_to(
  con,
  gapminder::gapminder,
  name = "gapminder",
  overwrite = TRUE,
  temporary = FALSE
)

# Get stuff out of it
gapminder_2007 <- tbl(con, I("gapminder")) |>
  filter(year == 2007) |>
  collect()

# All done
connections::connection_close(con)
```

### Espanso

[**Download Espanso**](https://espanso.org/) (macOS, Windows, and Linux)

[All my shortcuts and settings](https://github.com/andrewheiss/espanso)

Some things I do with it:

- Typographic things like:

  - `;comm` for ⌘, `;shift` for ⇧, `;opt` for ⌥
  - `;times` for a true multiplication sign ×, `;minus` for a true subtraction sign −
  - `;prime1` and `;prime2` for ′ and ″ (like 5′ 11″)

- …and date things like:

  - `;dt` for "2026-01-13"
  - `;dz` for "2026-01-13T11:06"
  - `;dm` for "2026-01-13 11:06:05"
  - `;date` for "January 13, 2026"

- …and longer words with repeated letters that I always mess up, like these:

  - `insttn` for "institution"
  - `consttal` for "constitutional"
  - `entrepal` for "entrepreneurial"

- …and code snippets like `;dplyrsum` for inserting this R code to disable {dplyr}'s grouping messages:

  ```r
  options(dplyr.summarise.inform = FALSE)
  ```

- …or running a {targets} pipeline with `;tarm`:

  ```r
  targets::tar_make()
  ```

- …or adding Quarto chunk metadata like disabling warnings and messages with `;wm`:

  ```yaml
  #| message: false
  ```

  ```yaml
  #| fig-width:
  ```
